# Admin tools to small-business mapping

This table maps current Sports Warehouse / Ability Links concepts to plain-language small-business equivalents. The purpose is to explain the Client Business Operating System without requiring clients to understand technical implementation details.

| Current concept | Small-business equivalent | Plain business value |
| --- | --- | --- |
| Public catalogue / storefront-style website | Customer-facing website | Customers can browse products, services, updates, and business information. |
| Admin dashboard | Private owner dashboard | The owner can inspect and manage business content without editing code. |
| ProductDB | Product or service catalogue | The business has one structured place for product or service details. |
| Product image manifest | Product/photo library | The business can track which products or services have approved images. |
| SourceRoot / image source folder | Approved photo library | Staff and the developer know where trusted images should come from. |
| Image management | Photo organization workflow | Images can be connected to the right products, services, pages, or updates. |
| Review approvals | Approve product or service content before publishing | The owner can approve, defer, or question changes before customers see them. |
| Gate reports | Ready/not-ready publication checklist | The business can see what is ready, incomplete, blocked, or needs review. |
| Safe publishing gates | Protected publishing process | Unfinished or risky changes are less likely to go live by accident. |
| Delivery blocked status | Not yet published / not yet ready | A clear status shows when something should not be shown publicly yet. |
| Flat CSV mirror | Spreadsheet-friendly review/export | The business can review information in a familiar spreadsheet format. |
| Review/tooling CSV mirrors | Simple audit and review files | Content can be checked outside the admin system when useful. |
| Read-only admin viewer | Business owner inspection page | The owner can safely inspect records without accidentally changing them. |
| Read-only manifest viewer | Photo status inspection page | The owner can review image readiness and evidence without editing files. |
| Source-of-truth records | Approved business records | Everyone can refer to the same trusted version of content or data. |
| Admin diagnostics | Website/business content health checks | The system can highlight missing content, broken assumptions, or areas needing attention. |
| AI-assisted documentation and workflow support | ChatGPT-supported maintenance guidance | The owner and developer can use AI help for descriptions, checklists, and training notes. |
| Codex and GitHub developer workflow | Developer-assisted technical delivery | Technical updates can be version-controlled, reviewed, and deployed safely. |

## Positioning note

The Website + Business Admin System should be explained in business terms first. Technical names can remain useful internally, but client conversations should focus on what the tool helps the business owner inspect, approve, maintain, and publish safely.
